# 개발일지 (DevLog)

## 폴더 규칙
```
DevLog/
├── 2026/
│   ├── 07/
│   │   ├── 2026-07-09.md
│   │   └── ...
│   └── 08/
│       └── ...
└── TEMPLATE.md
```
- 연도 → 월 → 일자별 파일로 관리
- 파일명은 항상 `YYYY-MM-DD.md`
- 작업한 날만 작성해도 무방

## 작성 팁
- 그날그날 GitHub 커밋 메시지를 그대로 붙여넣고 살을 붙이면 빠르게 작성 가능
- 막힌 부분(버그, 미해결 이슈)은 꼭 기록 → 다음날 이어서 보기 편함
- 월말에 `2026/07/SUMMARY.md`로 요약해두면 포트폴리오용으로도 활용 가능
